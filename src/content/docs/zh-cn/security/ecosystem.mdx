我们的Tauri组织生态系统托管在Github上, 提供了多项功能, 使我们的代码库更能抵御针对我们源代码和版本的攻击

为了降低风险并符合普遍采用的最佳做法,我们采取了一下方法

### 构建管道

我们在Github构建管道中发布源代码构建的过程高度自动化, 使用github Actions, 但仍需真人启动和审查.

### 签名提交

我们的核心代码库要求对提交内容进行签名, 以降低冒名顶替的风险, 并在检测到可能的破威胁后对归属的提交内容进行识别.

### 代码审查

所有合并到我们仓库的Pull Requests(PRs), 都需要至少一位项目维护者的批准, 在大多数情况下, 维护者就是工作组, 代码通常在PR中进行审查, 并运行默认的安全工作流程和检查, 以确保代码符合通用标准

### Release Process

我们的工作组审查代码更改, 标记 PR 的范围, 确保所有内容保持最新. 在发布次要和主要版本之前, 我们会努力对所有与安全相关的 PR 进行内部审核.

当需要发布新版本时, 其中一位维护者会在开着分支上标记新版本:

- 验证核心
- 运行测试
- 审核crates 和 npm 安全性
- 生成更新日志
- 创建构建产物
- 创建发布草案

然后维护者审核发布说明, 必要时进行编辑, 然后发布新版本.
